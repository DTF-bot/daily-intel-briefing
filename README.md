# 每日情报速递

每日情报速递是一个面向公开公司、产品和行业动态的本地 Web 情报助手。它可以按订阅配置定时抓取公开网页信息，生成中文情报报告，并把报告摘要推送到邮箱。

## 核心功能

- 配置情报订阅：名称、目标公司/产品、关键词、来源分类、限定域名、执行时间
- 手动运行一次：保存当前订阅后立即生成报告
- 定时运行：本地服务运行期间按每日时间自动执行
- 中文报告：按新闻、融资、产品发布、技术博客、行业动态等分类总结
- 邮箱推送：每个订阅可填写自己的 SMTP 和收件邮箱配置
- 历史记录：查看每次运行状态、报告链接、摘要和推送结果
- WebSocket 进度流：保留即时研究过程中的实时进度展示

## 合规边界

系统只处理公开可访问的信息：

- 不抓取登录后内容
- 不绕过验证码
- 不破解反爬机制
- 不抓取付费墙后的内容
- 不虚构来源链接
- 不确定的信息会在报告中标注“不确定”

## 本地运行

推荐使用一键启动脚本：

```powershell
.\start_daily_intel.bat
```

或者手动启动：

```powershell
.\.venv\Scripts\python.exe -m uvicorn backend.server.app:app --host 127.0.0.1 --port 8001
```

启动后访问：

```text
http://127.0.0.1:8001/config
```

## 配置说明

本项目不要把真实密钥写进代码仓库。请复制 `.env.example` 为 `.env`，在本地或服务器后台配置环境变量。

模型、搜索和邮箱配置优先通过页面表单填写；上传 GitHub 时不要上传：

- `.env`
- `data/*.json`
- `outputs/`
- `logs/`
- `.venv/`

这些内容已经在 `.gitignore` 中忽略。

## 定时任务部署建议

当前内置调度器只在 Web 服务运行期间生效，适合本地使用和个人服务器试运行。部署上线时有三种推荐方案：

1. 个人服务器：使用系统 cron 或 Windows 任务计划程序，每天固定时间调用后端“运行订阅”接口。
2. 单机长期运行：引入 APScheduler，并把任务状态、下一次运行时间、错过任务补跑记录持久化到 SQLite/PostgreSQL。
3. 多用户或生产环境：使用 Celery Beat + Redis/RabbitMQ，把调度、执行、失败重试和并发锁交给任务系统。

无论哪种方案，都需要保留这些能力：

- 任务持久化：服务重启后能恢复订阅。
- 错过补跑：服务停机期间错过的任务可选择补跑。
- 并发锁：同一个订阅同一时间只能运行一次。
- 失败重试：邮件推送失败不能影响报告生成。
- 运行详情：前端能看到当前阶段和失败原因。
